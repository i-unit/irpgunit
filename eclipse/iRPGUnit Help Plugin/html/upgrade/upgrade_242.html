Upgrading From RPGUnit v2.4.2 to v3.0.0

All your existing unit test suites should compile and run fine with iRPGUnit 3.0.0. However:

Test your existing test suites carefully in a sandbox environment, before upgrading to iRPGUnit 3.0.0.

The following test cases (compiled with v2.4.2) run fine with iRPGUnit:

  • CMDRUNLOGT
  • CMDRUNT
  • CRTTSTT
  • EXTTSTT
  • LIBLT
  • PGMMSGT
  • STRINGT

The following test cases (compiled with v2.4.2) do not successfully run with iRPGUnit:

  • Test cases testAssertWithFailure of test suite ASSERTT, because of internal changes.
  • Test cases testGoodByeIsNotHello of test suite ASSERTT, because of internal changes.
  • Test cases testTwoAndTwoIsNotEqualToFive of test suite ASSERTT, because of internal changes.

  • Test suite CALLPROCT, because service program RURUNNER has been dropped.
  • Test suite EXTPRCT, because service program RURUNNER has been dropped.
  • Test suite JOBLOGT, because service program RURUNNER has been dropped.
  • Test suite RUNT, because service program RURUNNER has been dropped.

  • Test suite RUACPTST, because source file RPGUNITF1 has been renamed to QTESTCASES. Some test cases succeed, when QTESTCASES is duplicated to RPGUNITF1. Other test cases still fail, because of different statement numbers due to internal changes.
The old RPGUNIT self-tests can not be compiled with version 3.0.0, because the source files have been renamed to more meaningful names.
