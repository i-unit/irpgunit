Introduction

The iRPGUnit plug-in connects the IBM Rational Developer for i with the RPGUnit library, that was started by Lacton back in September 2006. It adds a context menu item to the Remote Systems Explorer to let you run your RPGUnit test suites directly from the explorer.

The plug-in bases on source code originally developed by Mihael Schmidt at RPG Next Gen. It fixes a few bugs and adds some improvements, such as the report viewer and call stack information.

The RPGUnit library is an enhanced version of the RPGUnit library developed by Laction and continued by Cyril Clemenceau at RPGUnit at SourceForge. Unfortunately it seems as if the project is no longer alive. Nevertheless the original Internet pages at SourceForge are still a good source of information.

Important: Read the upgrade instructions when you consider to move to iRPGUnit.

Upgrading from v0.3.2 to 2.0.0
Upgrading from v1.10.0 to 2.0.0
Upgrading from v2.4.2 to 3.0.0

It is important to know that starting with v3.0.0 source member TESTCASE has been moved from source file RPGUNIT1 to source file QINCLUDE. Currently iRPGUnit is shipped with a proxy member in file RPGUNIT1 to ensure backward compatibility at compile time. This proxy member will be removed after 31.12.2019 with a later version of iRPGUnit.

Ensure to refactor your test suites in time!
